# Support & Issue Reporting

Thanks for using **Simple Mermaid Preview**! If you've found a bug, have a feature request, or need help, this guide will help you report it effectively.

---

## Before Opening an Issue

1. **Check existing issues** — search [open and closed issues](../../issues?q=) to see if it's already been reported.
2. **Make sure you're on the latest version** — update the extension from the Chrome Web Store and verify the issue still exists.

---

## How to Open an Issue

Go to [Issues](../../issues/new/choose) and pick the appropriate type:

### Bug Report

Please include:

- **Extension version** (visible in `chrome://extensions`)
- **Browser & version** (e.g. Chrome 123)
- **Website where it happened** (e.g. chat.openai.com, claude.ai)
- **What you did** — steps to reproduce the issue
- **What you expected** — what should have happened
- **What actually happened** — what went wrong
- **Screenshot or screen recording** — if applicable
- **The Mermaid diagram code** — paste the diagram that caused the problem (if relevant)

**Example:**
```
Extension version: 1.1.1
Browser: Chrome 123
Site: claude.ai

Steps:
1. Open a conversation with a mermaid diagram
2. The preview panel opens
3. Zooming in with scroll causes the diagram to disappear

Expected: Diagram stays visible while zooming
Actual: Diagram disappears after scroll zoom
```

---

### Feature Request

Please include:

- **What you want** — describe the feature clearly
- **Why it would be useful** — the problem it solves or the value it adds
- **Any alternatives you've considered**

---

### Question / Other

For general questions, open an issue with the **question** label and describe what you're trying to do.

---

## Tips for a Good Issue

- One issue per report — don't bundle multiple bugs or requests together
- Be specific — vague reports are hard to act on
- Include the Mermaid diagram code when relevant — this is the most helpful thing you can provide

---

Thank you for helping improve Simple Mermaid Preview!
